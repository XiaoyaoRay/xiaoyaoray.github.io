Hexo的NexT主题添加valine评论系统

发表于 2020-04-25 分类于 hexo 阅读次数： Valine： 2.3k 2 分钟

前言

Valine 诞生于2017年8月7日，是一款基于LeanCloud的快速、简洁且高效的无后端评论系统。

理论上支持但不限于静态博客，目前已有Hexo、Jekyll、Typecho、Hugo、Ghost 等博客程序在使用Valine

官网地址：https://valine.js.org/

LeanCloud上配置

请先登录或注册 LeanCloud

注意：注册时建议选择国际版，因为其他需要实名认证（身份证验证和支付宝验证）

获取APP ID 和 APP Key

进入控制台后点击创建应用，输入新应用名称并选择开发版，最后点击创建按钮即可

进入刚刚创建的应用，选择左下角的设置>应用Key，然后就能看到你的APP ID和APP Key了：

配置Web安全域

为了你的数据安全，请设置自己的安全域名。输入博客网站地址即可。

自定义邮箱

需要第三方插件Valine-Admin，配置后如果有评论会自动发送到邮箱。以下配置是我认为必不可少的，更多配置请访问插件官网。

在LeanCloud中部署插件

配置Git

选择左下角的云引擎>部署>部署项目>Git部署>配置Git

输入插件仓库地址https://github.com/zhaojun1998/Valine-Admin，点击保存按钮

部署插件

选择master分支，然后点击部署按钮部署插件

插件部署完成

配置环境变量

此外，你需要设置云引擎的环境变量以提供必要的信息，点击云引擎的设置页，设置如下环境变量。最后重启下应用。

必选参数

  • SITE_NAME : 网站名称。
  • SITE_URL : 网站地址, 最后不要加 / 。
  • SMTP_USER : SMTP 服务用户名，一般为邮箱地址。
  • SMTP_PASS : SMTP 密码，一般为授权码，而不是邮箱的登陆密码，请自行查询对应邮件服务商的获取方式
  • SMTP_SERVICE : 邮件服务提供商，支持 QQ、163、126、Gmail、"Yahoo"、...... ，全部支持请参考 : Nodemailer Supported services。
  • SENDER_NAME : 寄件人名称。

LeanCloud 自带定时器设置

免费版的 LeanCloud 容器，是有强制性休眠策略的，不能 24 小时运行：

  • 每天必须休眠 6 个小时
  • 30 分钟内没有外部请求，则休眠。
  • 休眠后如果有新的外部请求实例则马上启动（但激活时此次发送邮件会失败）
添加环境变量

首先需要添加环境变量，ADMIN_URL：Web 主机域名，如图所示（添加后重启容器才会生效）

配置定时器

然后点击云引擎 - 定时任务，新增定时器

  • 选择生成环境并选择self_wake
  • 添加Cron表达式：0 */20 7-23 * * ?

web 评论管理

web主机域名

首先确认 Web 主机的域名已配置，在云引擎>设置选择中查看，配置方式如下。

添加登录用户

后台登录需要账号密码，需要在这里设置，只需要填写 email、password、username，这三个字段即可, 使用 email 作为账号登陆即可。（为了安全考虑，此 email 必须为配置中的 SMTP_USER 或 TO_EMAIL， 否则不允许登录）

登录评论系统

在浏览器中输入web主机域名地址，然后输入存储中添加的email地址和密码登录系统。

NexT主题配置

打开在主题配置文件_config.yml并找到valine项。

  • enable设置为true
  • appid设置为LeanCloud上获取的值
  • appkey设置为LeanCloud上获取的值
  • notify如果配置了自定义邮件设置为true
  • language设置为zh-cn

其他配置请根据需求选择，怎么配置请参见NexT官网或者Valine官网
